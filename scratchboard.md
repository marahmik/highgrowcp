# 📋 HighGrow Schedule 시스템 스크래치보드
> 마지막 갱신: 2026-03-25

---

### 📝 프로젝트 구성 및 상태
1. **메인 페이지 (Main Page)** 
   - 서비스 소개 및 로그인/회원가입.
2. **마이 페이지 (My Page)**
   - 일반 직원이 로그인 후 마주하는 화면. 
   - 매장 목록 확인 및 소속 매장 관리. 퇴사 처리된 매장은 표시되지 않음.
3. **매장 페이지 (Store Page)**
   - **데스크톱**: 1~말일 한 화면에 출력되는 가로 그리드 (등급별 색상).
   - **모바일**: 주 단위(7열) 전통적 달력 뷰.
   - **접근 제한**: 퇴사자는 매장 캘린더 접근 시 차단 메시지 출력.
4. **관리자 페이지 (Admin Page)**
   - **직원 관리**, **매장 관리**, **통합 캘린더**.
   - **퇴사자 관리**: 퇴사일 지정 및 미래 스케줄 자동 삭제 기능 포함.
5. **근무 기록 (History Page)**
   - 개인별 과거 근무 기록을 확인할 수 있는 조회 전용 달력 페이지. 퇴사자도 본인 기록 확인 가능.

---

### ✅ 완료된 작업 내역

#### Phase 1~3 — 인류·기능 통합 & 모바일 최적화
- ✅ **매니저/직원 권한 분리**: 일반 직급은 본인 스케줄에 '요청'만 가능.
- ✅ **연차 관리**: 매니저가 각 직원의 연차 개수를 직접 수정 가능.
- ✅ **모바일 전용 7열 달력**: 주 단위 끊기 레이아웃 및 근무 유형별 색상 적용.
- ✅ **강력한 권한 시스템**: 전체 관리자/매니저의 타인 스케줄 수정 권한 강화.
- ✅ **미소속 인원 관리**: 어느 매장에도 소속되지 않은 인원 관리 기능.
- ✅ **근무기록 탭**: 개인별 조회 전용 7열 달력 구현 (퇴사자 접근 허용).

#### Phase 4 — 퇴사자 관리 및 보안 강화 🆕
- ✅ **퇴사자 전용 목록**: 직원관리 탭 내 활성 멤버와 퇴사자를 분리하여 관리.
- ✅ **퇴사일 지정 및 자동 정리**: 퇴사일 지정 시 해당 날짜 이후의 모든 스케줄 자동 삭제.
- ✅ **권한 자동 회수**: 퇴사자로 분류된 계정은 매장 캘린더 및 관리 메뉴 접근 권한을 즉시 상실.
- ✅ **본인 기록 조회 유지**: 퇴사 후에도 '근무기록' 탭을 통해 본인의 과거 데이터 확인 가능.
- ✅ **캘린더 노출 제외**: 모든 매장 캘린더 멤버 목록에서 퇴사자 자동 제외.
