# 쌀체듈러 (SSalcheduler) 기획 및 기능 정의서

초기 기획안(`docs/brainstorming/scratchboard.md`)과 최근 진행된 업데이트 내용을 종합한 최신 요구사항 및 기능 명세서입니다.

## 📌 핵심 목적
- 매장 파트타임/직원들의 월별 근무 스케줄을 효율적으로 관리하기 위한 웹 애플리케이션.
- 각 사용자(관리자, 직원 등)가 직접 로그인하여 권한과 직급에 따라 스케줄을 입력하고 관리할 수 있도록 구성.

## 👥 권한 및 주요 기능

### 1. 일반 사용자 (직원)
- **가입 및 매장 합류**: 이메일 또는 구글 계정으로 가입 후, 생성된 매장에 가입 요청을 보냄.
- **개인 스케줄 등록/수정**: 관리자 승인 후 소속된 매장 페이지에 입장하여, 월별 달력에 본인의 스케줄을 등록(근무/휴무 상태 선택).
- **마이페이지**: 가입된 매장 목록 확인 및 개별 매장 페이지 이동.

### 2. 관리자 (대표자 / Supervisor)
- **회원 관리 (멤버 관리)**:
  - 가입 요청 승인 및 권한 변경.
  - 멤버 이름 닉네임처럼 변경(Rename) 가능.
  - '매장 떠나기(Leave store / Transfer)' 와 '영구 삭제(Permanent delete)' 기능 분리 제공.
- **매장 관리**:
  - 다중 매장 생성 및 정보 수정 가능.
  - 매장별로 관리자만 작성 및 확인 가능한 '메모(Memo Box)' 기능.
- **통합 스케줄 관리 (기존 승인 탭 대체)**:
  - 복잡한 '결재/승인(Approvals)' 탭을 없애고 관리자 편의성을 위해 **[통합 캘린더 (All Schedules)]** 기능 도입.
  - **매장별로 분리 출력** — 각 매장의 스케줄을 개별 섹션(캘린더)으로 따로 보여주며, 각 매장별 단기알바 스케줄도 확인 가능.
  - 관리자 권한으로 직접 수정 및 삭제 가능.
  - 각 매장 캘린더 하단에 **매장별 메모** 개별 표시.
  - 슈퍼바이저(관리자) 뷰에서는 할당되지 않은 인원(Ghost slots)을 숨기거나 보이게 하는 커스텀 옵션 제공, 멤버 카운트 정렬 기능 지원.

## 📅 달력 및 스케줄 상태 정의
1월부터 12월까지 월 단위로 조회 및 작성. 요일, 근무 형태, 휴무일 등은 한눈에 구분이 쉽도록 직관적인 배경색으로 구분됨.

- **근무 형태**: `오픈`, `미들`, `마감`, `종일`
- **휴무 종류**: `연차`, `반차`, `대체휴`, `병가`, `요청`

## 🖥️ 페이지 구성

1. **메인 페이지 (Main Page)** 
   - 서비스 소개 및 로그인/회원가입.
   - 글로벌 워터마크/푸터 (HDH Hugo Kim & Jason Kim) 표시.
2. **마이 페이지 (My Page)**
   - 일반 직원이 로그인 후 마주하는 화면. 
   - 매장 목록 확인, 소속 매장 진입 혹은 미가입 시 매장 가입 요청.
3. **매장 페이지 (Store Page)**
   - 소속 인원들의 월별 달력이 표시되는 메인 작업 화면.
   - 다양한 기기 환경에서 깨짐이나 좌우 스크롤(가로 스크롤) 불편함이 없도록 **반응형(Adaptive) 디자인** 적용 (데스크톱 환경 포함, 한 화면에 1일~말일 출력 완료).
4. **관리자 페이지 (Admin Page)**
   - 관리자 권한을 가진 유저가 접근하는 통합 관리 대시보드.
   - **회원 관리**, **매장 관리**, **통합 캘린더(스케줄 전권 수정 및 뷰어)**로 구성.
   - 통합 캘린더는 **매장별 개별 섹션**으로 출력되며, 각 매장의 멤버 및 단기알바 스케줄을 각각 확인/수정 가능.
   - 각 매장 캘린더 아래에 **매장별 메모**가 개별 배치.

### ✅ 완료된 작업 내역 (최근)

#### Phase 1 — 레이아웃·반응형
- 데스크톱/랩탑에서의 15일 잘림 및 가로 스크롤 이슈 완전 해결 (1일~말일 한 화면에 꽉 차게 `table-fixed` 100% 적용).
- 달력 스케줄 칸(버튼) 내 불필요 패딩 제거 및 비율 조정으로 31일치 완벽 출력.
- Layout.tsx에서 `max-w-5xl` 해제 → 스케줄 페이지가 모니터 전체 너비 사용 가능. MyPage는 개별적으로 `max-w-5xl` 유지.
- ScheduleGrid에서 `sticky` 속성 제거 (full-width에서 불필요).

#### Phase 2 — 관리자 통합 캘린더·인증
- ✅ **관리자 통합 캘린더: 매장별 분리 출력 + 각 매장별 단기알바 스케줄 표시** — `AllSchedulesTab.tsx`에서 `StoreGroup[]` 기반 매장별 섹션 렌더링 완료.
- ✅ **통합 캘린더 하단 메모 박스: 매장별 분리 출력** — 각 매장 섹션 하단에 `handleMemoUpdate` 연동 완료.
- ✅ **로그인 시 "로딩 중"에서 안 넘어가는 현상** — `AuthProvider.tsx`의 `fetchProfile`에 `try/catch/finally` 적용 + 8초 safety timeout 추가로 해결.
- ✅ **Approvals(결재/승인) 탭 제거** → 통합 캘린더 탭으로 대체. `AdminPage.tsx`에서 `AllSchedulesTab`만 사용.
- ✅ **수퍼바이저 지점 UX** — 수퍼바이저 매장에는 단기알바 슬롯 숨김 + 안내 배너 표시.

### 🔧 다음 작업 / 검토 필요 사항

#### 🔴 우선 구현 (기능)
1. **월별 메모 시스템** — 매장 메모를 월(month) 단위로 저장·조회. 3월→4월 이동 시 해당 월의 메모가 표시되고, 다시 3월로 돌아가면 3월 메모가 유지됨. DB 스키마 변경 필요 (`store_memos` 테이블 신설 또는 `stores.memo`를 월별 JSON으로 전환).
2. **직급별 권한 분리** — 파트타이머/주니어/시니어는 **본인** 스케줄에 `요청` 티커만 설정 가능. 매니저 직급은 모든 티커를 모든 멤버에게 사용 가능.
3. **관리자 권한 범위** — 전체 관리자(admin)는 **관리자 페이지에서만** 모든 권한(매니저 이상) 행사 가능.
4. **탭 이름 변경** — 관리자 페이지 `회원관리` → `직원관리`로 변경.

#### 🟡 개선·리팩터링
5. ✅ **매장별 인원수 내림차순 정렬** — 통합 캘린더에서 이미 구현 완료 (`groups.sort(...)` in `AllSchedulesTab.tsx`).
6. **미사용 파일 정리** — `ApprovalsTab.tsx`가 코드에서 참조되지 않으나 파일이 아직 존재. 삭제 또는 아카이브 필요.
7. **전반적 UI/UX 개편** — 현재 기능 완성도는 높으나, 디자인 일관성·마이크로 인터랙션·테마 전환 시 UI 품질 등 전반 리팩터링 검토.
8. **모바일 최적화 강화** — 데스크톱 반응형은 완료되었으나, 모바일 환경(터치 인터랙션, 작은 화면 캘린더 표시 등) 추가 점검 필요.

#### 🟢 장기 과제
9. **에러 핸들링 고도화** — 네트워크 불안정 시 스케줄 저장 실패, Supabase 연결 끊김 등 엣지 케이스에 대한 사용자 피드백 및 재시도 로직 보강.
10. **성능 최적화** — 매장 수 증가 시 통합 캘린더의 `Promise.all` 4회 동시 쿼리 성능 모니터링 및 필요 시 페이징/lazy-load 도입.
11. **테스트 커버리지** — 핵심 로직(인증 플로우, 스케줄 UPSERT, 권한 체크 등)에 대한 유닛/통합 테스트 추가.
